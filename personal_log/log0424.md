## 实验方案讨论
### 方案一：便宜保守版

  目标是把固定每 8 步刷新改成“最长 16 步，但异常提前刷新”。只用已有 step 指标，不额外采样。

  参数：

  - step_since_slow
  - dp_ref_l2_ee6
  - aggregation_delta_ee6
  - jerk_l2_ee6
  - gripper_flip_count
  - dp_ref_l2_gripper

  触发函数：

  if step == 0:
      trigger slow

  if step_since_slow < 4:
      no trigger, except very severe gripper oscillation

  if step_since_slow >= 16:
      trigger slow

  risk =
      1[dp_ref_l2_ee6 > 0.035] +
      1[EMA3(aggregation_delta_ee6) > 0.18 or aggregation_delta_ee6 > 0.23] +
      1[EMA3(jerk_l2_ee6) > 0.32 or jerk_l2_ee6 > 0.42] +
      1[gripper_flip_count >= 2] +
      1[dp_ref_l2_gripper > 0.5]

  if step_since_slow >= 8 and risk >= 1:
      trigger slow

  if step_since_slow >= 4 and risk >= 2:
      trigger slow

  这些阈值大致对应当前数据的高分位：dp_ref_l2_ee6 的 p99 约 0.0406，aggregation_delta_ee6 的 p95/p97.5 约 0.175/0.227，jerk_l2_ee6 的
  p97.5/p99 约 0.31/0.42。这个方案最适合作为第一版，因为它简单、开销几乎为零，预计 slow call 能从 13% 降到大约 6%-9% 区间。

###  方案二：候选刷新点 veto + 少量不确定性检查

  目标更激进：不是每步判断是否叫 slow，而是在原本要 slow 的候选点决定“刷新还是跳过”。sample_var 不每步算，只在模棱两可时算。

  参数：

  - step_since_slow
  - dp_ref_l2_ee6
  - aggregation_delta_ee6
  - jerk_l2_ee6
  - gripper_flip_count
  - 二级检查：sample_var_ee6, sample_var_gripper

  触发函数：

  if step == 0:
      trigger slow

  if step_since_slow >= 16:
      trigger slow

  if step_since_slow >= 4 and (
      aggregation_delta_ee6 > 0.23 or
      jerk_l2_ee6 > 0.42 or
      dp_ref_l2_ee6 > 0.04 or
      gripper_flip_count >= 3
  ):
      trigger slow

  if step_since_slow in {8, 12}:
      cheap_score =
          1[dp_ref_l2_ee6 > 0.030] +
          1[aggregation_delta_ee6 > 0.050] +
          1[jerk_l2_ee6 > 0.300] +
          1[gripper_flip_count >= 2]

      if cheap_score >= 2:
          trigger slow

      if cheap_score == 0:
          skip slow

      if cheap_score == 1:
          run K=3 specialist uncertainty probe
          trigger slow if sample_var_ee6 > 5e-4 or sample_var_gripper > 0.08

  这个方案的关键是：sample_var 只作为“是否跳过 slow call”的二级裁决，不作为每步在线触发器。它每次约多 0.27s，但 slow system p50 约
  3.44s，所以只要它能避免一部分 generalist 调用就是划算的。

## 实验 先尝试方案二

### codex 建议的路线
 1. Age-based 空指导基线
      - slow 第一步必调。
      - age = step - last_slow_step
      - num_cond_actions = max(0, 8 - age)
      - age < 8 时按剩余 chunk 给 ref_action
      - age >= 8 时 ref_action = zeros，num_cond_actions = 0
      - max_age = 12 或 16 强制 slow
      - 这是最干净的检验：specialist 在没有新 generalist 指导时能撑多久。
  2. 方案二触发器 + 空指导
      - 在 age=8,12 做候选检查。
      - cheap score 低就继续空指导跳过。
      - cheap score 中等才跑 sample_var。
      - cheap score 高或 sample_var_ee6 > 5e-4 / sample_var_gripper > 0.08 触发 slow。
      - 这里 dp_ref_l2 在 num_cond_actions=0 时不再有原含义，应单独记录为 None 或只对有效部分算。
  3. 只作为对照：tail-hold / decay，不做线性外推
      - age >= 8 后把最后一个 generalist action 保持 1-4 步，或者按 alpha 衰减到 0。
      - gripper 只 hold sign，不做连续插值。
      - 不建议一开始做 a7 + k*(a7-a6) 这种线性外推，动作边界和 gripper 翻转都容易被放大。

  我会把实验优先级定为：空指导 max_age=12 → 空指导 max_age=16 → 触发器版 max_age=16 → tail-hold 对照。主要看每任务成功率、slow call
  rate、aggregation_delta_ee6、jerk_l2_ee6、gripper_flip_count，并把 age>=8 的 step 单独切出来分析。

### 实验注意事项
 Age-based 空指导基线 的核心不是马上做复杂触发器，而是先回答一个基础问题：generalist 给出的 8 步指导动作过期后，specialist 单靠当前观
  测、proprio、旧的 generalist latent hidden state，还能稳定撑几步？

  当前系统每次 slow call 会得到：

  - self.action: generalist 输出的 8 个指导动作。
  - self.hidden_states: OpenVLA 最后一层 action/token latent，作为 specialist 的全局条件。
  - specialist 每步仍然看当前图像、深度、夹爪图像、proprio。

  “空指导”只把 ref_action 置零，不是把 generalist 条件完全拿掉。因为 self.hidden_states 仍然会沿用上一次 slow call 的 latent。也就是说，
  这个实验实际测的是：

  > 当 slow system 不刷新时，旧 latent + 当前观测 + 空 ref_action 能不能让 specialist 继续控制。

  这比“插值伪造 generalist 输出”更干净。插值的问题是你只能外推 ref_action，但不能合理外推 hidden_states；这样容易制造一种假象：动作指导
  看起来连续，但 latent 语义其实已经过期。

  运行逻辑
  用 age = step - last_slow_step 表示距离上次 generalist 调用过了几步。

  假设 max_slow_age = 12：

  step 0:  调 generalist，age=0，ref_action 有 8 个有效动作
  age 1:   ref_action 剩 7 个有效动作
  age 2:   ref_action 剩 6 个有效动作
  ...
  age 7:   ref_action 剩 1 个有效动作
  age 8:   ref_action 全零，进入空指导
  age 9:   ref_action 全零
  age 10:  ref_action 全零
  age 11:  ref_action 全零
  age 12:  强制调 generalist，重新开始

  如果 max_slow_age = 16，就会有 age=8..15 共 8 步空指导。max_slow_age = 8 则没有空指导，可作为 sanity check。

  这里建议定义两个参数：

  - empty_ref_after_age = 8：8 步 chunk 用完后开始空指导。
  - max_slow_age = 12/16：最多允许 stale generalist latent 用多久。

  为什么必须改 ref_action 对齐
  现在代码里的 slow call 和 ref_action 都硬绑全局 % 8。

  关键位置在 vla-scripts/dual_sys_evaluation.py:193：

  if (step + 1) % 8 == 0 or step == 0:
      ...

  以及 vla-scripts/dual_sys_evaluation.py:207：

  num_cond_actions = 8 - (step + 1) % 8

  如果只跳过 slow call，不改 num_cond_actions，代码会把旧 chunk 当成新 chunk 重新对齐使用。这是错误实验。动态调度必须改成基于 age，不是
  基于全局 step modulo。
